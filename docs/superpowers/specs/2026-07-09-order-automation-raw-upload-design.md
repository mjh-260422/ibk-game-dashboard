# 주문 자동화 - 로우데이터 파일 업로드 기능 설계

## 개요
`업무_우리카드구간포상/주문_자동화_script.js` (실물배송 자동화, 스프레드시트 ID `1Ce3swcsiFJLRW9dnAqnvXjmPmh2istTASiHOwCbnUiU`)에 "로우데이터 여러 파일을 앱스크립트에 첨부해서 자동 취합" 기능을 추가한다.

## 배경 / 현재 방식
사용법 탭에 명시된 현재 프로세스:
1. 크라스에서 배송지 입력 완료 건 조회 → 엑셀(.xls) 다운로드 → 전체 복사
2. "로우" 탭 클릭 → 기존 내용 전체 삭제 → 붙여넣기(값만)
3. 메뉴 "주문 자동화 > 변환 & 파일 저장" 실행

크라스 다운로드가 여러 파일로 나뉘는 경우(예: `0529/배송 로우/관리용_*.xls` 4개) 수동으로 취합해야 하는 번거로움이 있음.

## 목표
- 여러 개의 `.xls`/`.xlsx` 로우데이터 파일을 앱스크립트 다이얼로그에 첨부하면 자동으로 취합하여 "로우" 탭에 채움
- 시트 편집 권한만 있으면 별도 설정(Drive API 등) 없이 누구나 사용 가능해야 함 (범용성)
- 데이터 누적 없음 — 매 실행마다 "로우" 탭 전체를 새 데이터로 대체
- "변환 & 파일 저장"(주문 자동화 실행)은 이 기능과 분리 — 업로드는 "로우" 탭 채우기까지만 담당, 실행은 기존처럼 메뉴에서 별도 클릭

## 비목표
- 병합 시 헤더 불일치 파일을 건너뛰고 나머지만 처리하는 기능 (→ 헤더 불일치 시 전체 중단)
- "변환 & 파일 저장" 자동 연쇄 실행
- 기존 CSV 업로드(`Code.gs`/`Upload.html`, 리마인드 발송용)와의 통합 — 완전히 별도의 기능/파일로 추가

## 접근 방식
브라우저 측에서 SheetJS(`xlsx.full.min.js`, CDN)로 `.xls`/`.xlsx` 파일을 직접 파싱해 JSON으로 서버에 전달한다. Google Drive API 변환을 쓰지 않으므로 Apps Script 프로젝트의 고급 서비스 활성화나 추가 OAuth 스코프가 필요 없다 — 시트 열람/편집 권한이 있는 사람이면 그대로 사용 가능.

(대안으로 검토했던 Drive API 변환 방식은 임시 파일 생성/삭제와 고급 서비스 활성화가 필요해 범용성 요구와 맞지 않아 제외.)

## UI / 진입점
- `주문_자동화_script.js`의 `onOpen()` 메뉴("주문 자동화")에 항목 추가: **"로우데이터 업로드"** (기존 "변환 & 파일 저장" 위에 배치)
- 클릭 시 모달 다이얼로그(`RawUpload.html`, 신규 파일) 표시
- 다이얼로그 구성 (기존 `apps_script/Upload.html`과 동일한 톤의 드래그앤드롭 UI 재사용):
  - 파일 선택 영역 (`.xls,.xlsx`, multiple)
  - 선택된 파일 목록 표시
  - 안내 문구: "선택한 파일로 '로우' 탭의 기존 내용을 모두 대체합니다"
  - 확인 체크박스 (체크 전에는 업로드 버튼 비활성화) — 기존 스프레드시트 전체 덮어쓰기 확인 체크박스 관례를 따름
  - 업로드 버튼, 상태/에러 메시지 영역

## 데이터 흐름
1. 사용자가 파일 선택 → 각 파일을 `FileReader.readAsArrayBuffer`로 읽음
2. `XLSX.read(arrayBuffer, {type:'array'})`로 워크북 파싱 → 첫 번째 시트를 `XLSX.utils.sheet_to_json(sheet, {header:1, raw:false, defval:''})`로 2차원 배열 변환 (헤더 포함)
3. 모든 파일의 `{name, rows}` 배열을 모아 `google.script.run.processRawFiles(filesData)` 호출
4. 서버(`processRawFiles`):
   - 파일이 0개면 에러 반환
   - 각 파일의 첫 행(헤더)을 문자열 배열로 trim 비교. 파일 간 헤더가 정확히 일치하지 않으면 즉시 실패 반환 (어느 파일이 어떤 컬럼에서 다른지 메시지에 포함) — "로우" 탭은 건드리지 않음
   - 헤더가 모두 일치하면: 헤더 1행 + 모든 파일의 데이터행(2행부터)을 파일 순서대로 이어붙임
   - "로우" 탭 기존 내용 전체 삭제 후 병합 데이터 `setValues`로 기록
   - 성공 메시지: 파일별 행 수 + 총 병합 행 수
5. 다이얼로그에 결과 표시, 성공 시 사용자가 시트로 돌아가 "변환 & 파일 저장"을 별도로 실행

## 에러 처리
- 파일 파싱 실패(손상된 파일 등): 해당 파일명과 함께 에러 표시, 서버 호출 자체를 막음 (클라이언트 단에서 처리)
- 헤더 불일치: 서버에서 즉시 실패 반환, "로우" 탭 미변경
- 빈 파일(데이터행 0개): 헤더만 있는 파일은 경고 없이 포함(0행 기여) — 전체가 비면 에러
