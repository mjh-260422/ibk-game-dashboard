현재 구조는 다음과 같이 이해하고 있습니다.
게임 목록 페이지(윈큐브 마케팅)
→ 게임 실행 페이지(두잇파이브)
→ 보관함 페이지(윈큐브 마케팅)
→ 뒤로가기 클릭 시
→ 게임 실행 페이지로 이동

현재 희망하는 프로세스는 보관함 페이지에서 뒤로가기를 눌렀을 때 게임 실행 페이지가 아니라 바로 게임 목록 페이지로 이동하도록 하는 것입니다.
이를 위해서는 게임 실행 페이지에서 보관함 페이지로 이동하기 전에, 브라우저 history 스택에서 게임 실행 페이지 이력을 제거하는 방식이 필요할 것으로 보입니다.
Next.js애서는 router.replace를 사용하여 현재 페이지를 history에 남기지 않고 이동하는 처리가 가능합니다.
혹시 PHP에서도 이와 유사하게 이전 페이지 이력을 남기지 않도록 처리하는 것이 기술적으로 가능한지 확인 부탁드립니다.

유선으로 안내드린 내용을 다시 정리하여 전달드립니다.

기존에는 게임 실행 전에 포인트를 먼저 차감한 후 두잇 파이브로 인입하는 방식으로 안내드렸습니다.
다만, 이 경우 네트워크 문제나 게임 실행 중 오류가 발생했을 때 포인트 복구 처리 및 CS(고객 문의) 발생 가능성이 있어, 처리 방식을 아래와 같이 변경하게 되었습니다.

게임 실행 시점에 저희가 제공하는 보상 안내 API를 호출하는 시점에 포인트를 차감하는 방식으로 진행 부탁드립니다.
해당 변경으로 인해 기존에 제공드린 API 스펙 자체에는 변경 사항이 없습니다.

또한, 게임 목록에서 특정 게임을 선택할 경우에는
현재 보유 포인트로 게임 실행이 가능한지에 대한 사전 유효성 체크를 진행한 후 두잇 파이브로 이동하게 됩니다.

정리하면, 게임은 정상적으로 실행된 이후에 보상 안내 API를 호출해 주시고,해당 API의 응답 값을 기준으로 아래와 같이 처리해 주시면 됩니다.

당첨 시: 당첨 상품 노출
오류 발생 시: "오류가 발생했습니다", "포인트가 부족합니다" 등의 안내 메시지 노출

위와 같은 방식으로 구현 부탁드립니다.
